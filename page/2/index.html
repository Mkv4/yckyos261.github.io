遇见狗瘟与治愈猫瘟

2017-05-01

狗瘟

在我家那种偏远地区，狗和猫都是直接送的，谁家生小猫小狗了可以去直接抱的。

小时候我家里养了一只狗，每次我回家它就会舔我脸，那个时候，我还没有它高。后来它被别人家的耗子药毒死了，我哭了很久很久。

我非常喜欢狗。在读大学的时候就梦想要是能养一只狗就好了。

我从来没有想过狗和猫可以卖的这么贵，难道这也是有阶级的？

我第一次看见一只金毛要三千多我就很震惊，当然这不算最贵的。我感觉我养狗的希望快要落空了。

很明显，我没有那么多钱，只好看看淘宝，寄希望会有一些便宜的。果然淘宝上才300块这么便宜。

赶紧问商家有没有狗。一问自然是有的。让我到哪个地方去买狗到了告诉他（淘宝的客服）。到了指定地点却没有看见有宠物店，他让我等下，结果等了一会是另外一个人号码给我打电话。问我是不是要买狗，然后指挥我去什么什么地方，花了1500买了一只金毛。

具体我真是不想多说。反正结果就是买到一只病狗，有犬瘟细小。上网一查那个地方有非常多的人在哪里买到的狗都是病狗。连新闻都报道过了。但是他还是好好的开着。

我不知道当时我请假去找他算账的时候打了多少个110，多少个12345，还有新闻热线。但是结果哦呵呵的。我也不敢多说。

我肯定不会放弃，我查了查，他们这种诈骗方式在网上已经成规模了，有成熟的套路，从病狗是哪里来的，教你怎么说怎么卖。

比如他们都说是寄卖，寄卖协议，领养协议，工商只会问有没有发票啊？？没有发票不管。 从各种环节上来设局等你上套。

老子还就不信了，去法院告他。写申请准备材料都是我自己做的。材料不合格，回来改了一次，去了，交了五十块钱立案，案由是消法第四十九条，那些日子我看了很多知识，想着一定要告赢他。

后来我回家了，法院打电话告诉我来吧啥啥啥时候开庭，我说我不在我让别人来行吗，他说那么多天你还回不来吗。我家是偏远山区来一趟杭州挺贵的。

后来我也没去了，心累了，绝望。

我给狗取了个名字叫数学，数学死的那天晚上哀嚎了两声，我却无能为力。

在这之前我根本不知道这世界上有星期狗一说。人到底能有多坏？

我应该是怪我没钱贪便宜才害了数学吧。没治好它。

反正至今那25块钱也没有退给我。

猫瘟

我主要是想讲述一下没钱的人怎么治猫瘟。

这只猫是在数学之后几天带回家的，它是从个人家里买的，它的名字叫英语。

所以买来之后的每一针疫苗我都有打。

我以为打了疫苗就不会再得病了，哪知道这打了疫苗还是会得猫瘟。

在这货八个月的时候搬了一次家，之后过了几天他就吐两次，还不吃东西了。刚开始带去医院检查，拍片、抽血，我问医生为什么要拍两个片子呢，她说位置不同，但最后拍出来的片子只用了一张，另外一张她看都没看。

也没有查出什么来，只是抽血说白细胞低。

她还要继续往下检查，但是出于拍片的不信任，我没有让她继续检查。

回家过了几天英语已经吐得比较频繁了。我才怀疑是不是得猫瘟了，上网查了查症状，用以前买的猫瘟试纸测了一下，测出来的弱阳性。拿着试纸去医院问医生。

医院当然是说你立马带猫过来住院吧，很明显我没钱。有钱也不愿意给他们坑。

所以就自己去搜了一些论文上淘宝去买了药。连打了一周。其实针打到第三天这个猫就活蹦乱跳的了。第一天打完它就不吐了。

大多数的方法都跟这里面写的是差不多的。

我总结了一下，这个猫为啥活过来了。

  • 它比较老了，没小猫那么脆弱。
  • 打针及时，没有拖到拉血。
  • 最主要是人不要放弃，不愿意被医院坑，也不要放弃治疗。
  • 知识改变命运、信息获取的渠道很关键。

Bloom Filter

1、在一个很长的二进制向量（可以设置为一个一维数组，但定义中是bit位）中把所有位置都初始化为0；

2、假设有一个集合有S{x,y,z}三个元素，还有三个hash函数；

3、分别用hash1(x),hash2(x),hash3(x),去计算x,y,z，这三个元素。

4、得出来的结果会对应1中向量的某个位置，把对应的位置设置为1。

  • 查询
    如果要查询结果是否重复则把待查询的数据，分别用相同的hash函数去计算结果，去1中查找对应位置 查看数据是否为1，如果像例子中的三个位置都查询到结果为1，那么就可以判断待查询结果已经存在。

MD5去重树

(1) 将url进行16位的MD5压缩。MD5加密算法有16位和32位两种方式理论上来说16位的MD5加密的碰撞概率为1/264。因而不同的url生成的MD5密文相同的可能性非常之小这就保证了去重效果。虽然32位的去重效果更佳好,但却比16位的多耗一倍的空间,因此我们采用16位的数据加密;

基于正文结构树的去重方法

正文的物理结构是对正文内容的一种层次划分，可以将其表示为一棵树的形式。其中标题为根节点，每个自然段按照其在全文结构中所处的层次表示为树的各层节点。由于段落内容通常较长，所以可以使用段落编号代替段落本身来生成正文结构树。段落编号是将正文中的各个自然段进行顺序编号，其中正文标题标号为零。这样既节省了存储空间也提高了建树效率。对正文结构树进行先序遍历将得到正文段落的顺序编号，层次遍历将得到正文的一个层次划分。

每个标签赋予一定的权值，如h1标签

小标题编号、HTML 标签和短段落标签的权重按照从高到低排列。如果多个类型标签用在同一篇文章中，则按照较高权值对其标识。

  • 正文结构树生成流程
    1 扫描正文(这里的正文指经过噪音去除并保留周围 HTML 标签的网页正文)，生成段落 信息表;

2 分析段落信息表，使用正文结构表现形式获取算法获取正文结构表现形式;

3 根据结构表现形式，使用权值分配算法对正文中的每个自然段赋权值;

4 使用正文结构树建树算法生成相应的正文结构树。

  • 建树算法(extrcatlg 算法)

令T={I0,I1,I2,…,In}表示网页正文，其中I0为标题编号，Ii为正文第i自然段的编号。W={W0,W1,W2,…,Wn}为自然段对应的权值。网页正文可以按如下规则转化为一棵正文结构树。

1若T为空，StrucTree为空;

2若T非空，则令I0为正文结构树的根节点，I1为结构树的第一个孩子节点。后面的自然段编号按照其权值大小进行插入操作。

插入操作遵行以下规则:

规则1:如果Wi+1=Wi，则Ii+1作为Ii的右兄弟节点。

规则2:如果Wi+1<Wi，则Ii+1作为Ii的孩子节点。

规则3:如果Wi+1>Wi，则继续跟Ii的父节点比较。比较结果如果满足以上两个规则，则按照这两个规则进行递归操作。如果Ii的父节点为根节点，则插入一个节点I作为Ii的左兄弟节点，令其节点对应内容为“总括段”，将Ii+1作为I的子节点。

3使用上述三个规则对段落编号依次进行操作，直到找到合适的位置将每个段落编号都插入到正文结构树中。

改进

如果段落中存在某句、或者某词语存在重点标注的特点，那么则认定此句子或者词语的存在更为重要，其所在段落重要性尤为突出。这种现象表明了内容块的嵌套问题，即内层的的重要标签的权值可以像其所在内容块的外层标签权值传递(内层标签权值>外层标签权值)。基于这个理论，如果段落中存在部分重要信息，那么其HTML标签权值按照相应算法依次向父节点、兄弟节点传递。

如图，叶子节点中的一个段落的部分句子、词语出现了基于强调的HTML标签，则将此标签权值赋予整个段落，其父节点、兄弟节点的段落权值依次改变，从而提高各个段落的影响因子。经过对重复网页特点地分析，转载工具为了反去重，不仅会对网页文本内容进行修改，而且也存在对网页正文格式做改动。例如上图所示。所以，此种嵌套标签权值传递方式仅适合结构没有受到丝毫改变的重复网页，导致大部分近似重复网页漏检。

规则1:当检测到某个段落存在多组标签，并且各组开始标签之间、结束标签之间都没有其它中文字符，表明此段落由多组标签格式定义，而且全段落格式一致，则按照标签最大值对此段落赋权值。

规则2:某一段落中有多组标签，但这些标签并不是用于此段落的所有文字，则按照检测到的最小标签权值为本段落赋权值。

规则3:正文结构树进行匹配时，并不按照标签权值比较两棵树，仅是按照已经建立好的树的层次进行比较，此时忽略掉标签权值。

阿里巴巴JAVA开发手册记录

2017-04-12

我摘录了一些放在别的语言里面也能用的规则。确实很精华。。
我看了看那些这牛那牛写的代码真是连最基本的可读性都没有。

不要用_开始
类名使用 UpperCamelCase 风格，必须遵从驼峰形式
常量要定义，常量可以放在一个类中。要规定好类；
左大括号之前不换行，后换行，右大括号之前换行；右大括号之后有else等不换行。
左括号和后一个字符之间不出现空格;同样，右括号和前一个字符之间也不出现空 格。
if/for/while/switch/do 等保留字与左右括号之间都必须加空格。
任何运算符左右必须加一个空格。
方法参数在定义和传入时，多个参数逗号后边必须加空格
线程资源必须通过线程池提供，不允许在应用中自行显式创建线程
高并发时，同步调用应该去考量锁的性能损耗。能用无锁数据结构，就不要用锁;能 锁区块，就不要锁整个方法体;能用对象锁，就不要用类锁。
对多个资源、数据库表、对象同时加锁时，需要保持一致的加锁顺序，否则可能会造 成死锁。
创建线程或线程池时请指定有意义的线程名称，方便出错时回溯。
在一个 switch 块内，每个 case 要么通过 break/return 来终止，要么注释说明程序 将继续执行到哪一个 case 为止;在一个 switch 块内，都必须包含一个 default 语句并且放在 最后，即使它什么代码也没有。
在 if/else/for/while/do 语句中必须使用大括号，即使只有一行代码，避免使用下 面的形式:if (condition) statements;
推荐尽量少用 else， if-else 的方式可以改写成:
循环体中的语句要考量性能，以下操作尽量移至循环体外处理，如定义对象、变量、 获取数据库连接，进行不必要的 try-catch 操作(这个 try-catch 是否可以移至循环体外)。
在使用正则表达式时，利用好其预编译功能，可以有效加快正则匹配速度。 说明:不要在方法体内定义:Pattern pattern = Pattern.compile(规则);
异常不要用来做流程控制，条件控制，因为异常的处理效率比条件分支低。
对大段代码进行 try-catch，这是不负责任的表现。catch 时请分清稳定代码和非稳 定代码，稳定代码指的是无论如何不会出错的代码。对于非稳定代码的 catch 尽可能进行区分 异常类型，再做对应的异常处理。
捕获异常是为了处理它，不要捕获了却什么都不处理而抛弃之，如果不想处理它，请 将该异常抛给它的调用者。最外层的业务使用者，必须处理异常，将其转化为用户可以理解的 内容。
可被用户直接访问的功能必须进行权限控制校验。
用户敏感数据禁止直接展示，必须对展示数据脱敏
用户输入的 SQL 参数严格使用参数绑定或者 METADATA 字段值限定，防止 SQL 注入，
禁止字符串拼接 SQL 访问数据库
用户请求传入的任何参数必须做有效性验证。
禁止向 HTML 页面输出未经安全过滤或未正确转义的用户数据。
表单、AJAX 提交必须执行 CSRF 安全过滤。
URL 外部重定向传入的目标地址必须执行白名单过滤。

木马实现技术概述

2017-03-30

概述

木马应该带有哪些功能？我能想到一些。

  • 获取权限、修改安全策略、修改用户组
  • 获取系统信息、网络信息。
  • 端口映射、网络嗅探、对内网渗透的支持如socks转发功能。
  • 进程隐藏、网络连接隐藏、文件隐藏、进程管理
  • 键盘记录、下载传输文件、流量隐藏、对抗杀软、取证工具。

细分一下，木马需要自启动，处理文件需要下载、上传、创建、显示目标文件、执行指定文件，显示文件的界面中的图标需要跟windows系统上自带的图片相契合。可能还需要监控屏幕、录音。

但是一个木马应该要解决的首要问题应当是活下来；之后才能考虑完成任务。那么要活下来最关键的需要的能力是什么？隐藏自己。

进程隐藏

进程伪隐藏

目前不太适用，把木马进程注册为服务，通过系统的服务器管理启动。进程ID任然会存在。任务管理器中的服务会显示出来。

动态链接库隐藏

自身的计算机进程隐藏起来；，举个例子，
HOOK API。程序 a 会调用一个api，hookapi就是当程序执行到一个call 进入到某个api后，修改api的起始地址，然后跳转到你的代码，执行，之后再跳回 api，api脱钩之后跳回恶意代码，最后恶意代码执行完毕之后跳回原本程序a。（其实这里可能会用到dll注入。）
一般使用全局钩取。钩取Ntdll.ZwResumeThread() API,之后运行的进程就会自动注入你的dll。注入dll不会产生进程。

这样的方法有个问题，对cpu依赖比较大，cpu的jump指令可能不同。还有就是，资源竞争的情况下，同时竞争改写函数（即修改的api）。可能会出问题。

还有就是通过改变IAT表中的函数地址来hook。

远程线程注入

  • 创建远程线程，CreateRemoteThread() API,

  • 使用注册表的AppInit_DLLs

实现你要的功能。
此外还有些方式：

  • 利用注册表隐蔽启动，如[HKLM\Software\Microsoft\Windows\CurrentVersion\Run]
    [HKLM\SYSTEM\ControlSet001\Control\SessionManager\KnownDLLs] KnownDLLs 子键存放着一些已知 DLL 的默认路径，修改路径可以达到加载恶意dll的目的。

  • 利用注册表文件关联项进行隐蔽启动
    HKEY_CLASSES_ROOT 和 HKLM\Software\CLASSES 目录下包含的子文件夹对应一种文件类型，修改即可达到关联启动的目的。

如，
[HKCR\textfile\shell\open\command] @=” notepad.exe %1\” %“ [HKLM\Software\CLASSES\textfile\shell\open\command]@=”notepad.exe%1\” %“ 如果将其改为 “notepad.exe 木马.exe %1 %*”，木马程序将在每次打开文本文件时调用 notepad.exe 文件后被执行。

隐蔽通信

可能对方的网络中会装有各种防火墙或防护设施监控当前流量。或者管理员手工检测分析。

反向链接技术

顾名思义，不需要你控制端去连接被控端，被控端主动来连接控制端。大多数木马使用的都是这样的技术。

现在木马比较喜欢用ICMP协议，因为不用打开通信端口。

这里主要想说一下内网如何穿透出来。目前业界比较流行的方法是用UDP、DNS来绕过防火墙等。

有一台公网服务器a，内网b 黑客 c。

黑客想要连上木马，需要通过内网的b去链接公网服务器a，之后a转发数据到黑客c。这样就完成了一次通信。

那么问题来了，怎么携带数据出去？

内网中的NAT穿透

现在比较流行的是利用DNS查询请求来携带数据，因为DNS的包是UDP，并且端口不容易被封。其实真正的木马除了53端口还会使用很多常见的端口。

dns实现概述，自写一个server放在公网上接受dns查询，提取数据包；做转发。

需要是现实的点有:标记源ip（内部） 源端口（内部） 目的IP 目的端口，要访问的目标ip（需要服务端做转发的）。因为穿透NAT的时候会做映射转换，要考虑完全锥形、受限锥形等问题；我以前写过这个问题，博客里有。UDP的。

  • 内部主机 A 向拥有公网 IP 的第 3 方服务器 S 发送一 个 UDP 包,NAT 设备 NA
    记录了这次会话,一个由 A 的内部 IP 及端口号到 NA 的外部 IP 及端口号的映射被建立
    (设 A 经过 转换后的公网 IP 及端口号为 EIPA:EPORTA), 此时 S 可以与 A
    进行通信了。

  • (2)同理,B 向 S 发送一个 UDP 包,(设 B 经过转换后的公网 IP 及端口号为EIPB:EPORTB),此时,S 亦可以与 B 进行通信了。

  • (3)当 S 收到 A 或 B的协助连接申请后,分别将 A、B 经过 转换后的公网 IP 地址及端口号EIPA:EPORTA,EIPB:EPORTB 分别 发送给对方(将 A 的地址告知 B,将 B 的地址告知 A）

  • (4)A、B 互相发包连接。不妨假设 A 先向 B 经过转换后的 地址 EIPB:EPORTB 发 UDP包,若 NB 是完全锥型 NAT,该包会 被转发至 A,通信成功;但大多数的 NAT是受限型的,这时该 包虽然会被 NB 丢弃(因为该包是不请自来的),但已经成功在 NA上打了一个到 EIPB:EPORTB 的洞。 此时 B 向 EIPA:EPORTA 发 送 UDP 包, 由于 NA上已经被打上了到 EIPB:EPORTB 的洞,所 以即使是受限型的 NAT也会认可这个包的合法性, 从而将该包转发至私网内的 A。

端口复用

端口复用首先需要判断端口是否打开，如果打开再进行下一步操作，需要判断发过来的数据包是否是给木马的，如果是，可以转发给木马。如果不是直接放行，这样的话可以在木马的数据包中加一个标记即可识别。

通信隐蔽

很明显——加密，但是不建议使用base64这类的加密，实在是有点明显。现在的c&c有一个特点都是去请求某一个网址或者是Facebook，twitter上的某些文章。这样可以避免追踪。

当被控端连接控制端之后，控制端向被控端发送数据，可以伪装目的ip和源ip；这样被控端只要嗅探一下就知道是控制端发来的消息。从而隐藏了自己。

文件隐藏

木马的自删除

复制源程序到指定目录，检测是否复制成功，成功之后删除程序，否则再复制。

属性修改

将自身的属性修改为隐藏，或者把自己放在某个隐藏文件之内。

DLL替换

用恶意的dll替换系统原本的dll，当然原本的dll的功能是需要保留的。

其他

你可能需要一些守护进程来守护你的钩子。像NSA一样做一些非常猛的插件化的东西，这样既可以躲避杀软又可以增加程序的可扩展性。
